Deploying Angular app in gh-pages

this page covers #Angular #deploymnts using #gh-pages for Angular applications only

GitHub Pages is designed to host your personal, organization, or project pages from a GitHub repository. We can deploy the repositories, applications or static pages for free and it is simple to use, just paste your files or commit to the branch and choose a desired branch to deploy securely now it is enforce HTTPS enabled, which provides a layer of encryption that prevents others from snooping on or tampering with traffic to your site. When HTTPS is enforced, your site will only be served over HTTPS.

We might already know angular-cli as a tool for Angular apps and what it can do. Here we use angular-cli-ghpages to deploy our Angular application.

First get in to your project directory cd yourProjectDirectory then Install angular-cli-ghpages  -g means globally

Now use the angular-cli build cmd we use in angular application to produce target folder with --base-href flag here "repo-name" should be https://userName.github.io/yourRepoName/ only /yourRepoName/ cause when the dist gets deployed, the index file looks for the assets and other files to render, so many other pages may have mentioned otherwise but trust me the pages will not reload properly, So follow the steps and by the end of this tutorial you will understand on your own.

After the above cmd, there should be dist created with all the assets in your project

ngh is the shorthand to publish your dist folder to gh-pages branch in your repository

Go to your repository settings and you will see the deployed application link
